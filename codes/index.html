Setting up

Posting Apache Config File.htaccess to root directory

  • log in to Hostinger, websites > dashboard > file manager > public_html
  • if you already have a .htaccess file, right-click and choose Edit.
  • if you do not have a .htaccess file, click New file on the left panel
  • paste the following code: 
    <IfModule mod_headers.c>
# Revalidate HTML/PHP on each request
<FilesMatch "\.(?:html?|php)$">
  Header set Cache-Control "no-cache, no-store, must-revalidate"
  Header set Pragma "no-cache"
  Header set Expires "0"
</FilesMatch>

# Long-cache versioned assets
<FilesMatch "\.(?:css|js|png|jpg|jpeg|gif|webp|svg|woff2?)$">
  Header set Cache-Control "public, max-age=31536000, immutable"
</FilesMatch>
</IfModule>

# Optional: tidy up ETags
FileETag None
Header unset ETag

    the code tells browsers to always fetch a fresh copy of your HTML/PHP pages but to store your static assets (like images, CSS, and JavaScript) for one full year.

Version your assets

  • this is the most reliable method. Add version numbers or timestamps to all CSS/JS files as shown below:: 
    <link href="../assets/css/codes.css?v=251008" rel="stylesheet" />
<script src="../assets/js/codes.js?v=251008" defer></script>

Clear Hostinger's Server Cache

  • If you still see your old content after uploading your changes with a new version number, you need to manually clear Hostinger's cache in hPanel:
  • log in to Hostinger, websites > dashboard > clear cache
  • Cache will be removed from caching plugin, Hostinger CDM, and server side cache.
